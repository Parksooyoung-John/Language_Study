============================================================
1. 프로젝트 배경
============================================================

이 프로젝트의 출발점은 개인적인 학습 불편이었다.

중국어 HSKK 말하기를 준비하려면 매일 실제로 말해야 한다.
하지만 혼자 공부하면 말하기 연습까지 가기 전에 많은 마찰이 생긴다.

기존 학습의 문제:
- 오늘 무엇을 공부할지 매번 정해야 한다.
- HSKK 유형에 맞는 질문과 답변을 직접 찾아야 한다.
- 한자, 병음, 한국어 뜻을 같이 정리해야 한다.
- 실제로 말해 볼 기회가 부족하다.
- 녹음하고 다시 들어보는 흐름이 번거롭다.
- 답변이 맞는지, 더 자연스러운지 바로 알기 어렵다.
- 피드백이 있어도 다음 학습으로 체계적으로 연결되지 않는다.
- 범용 회화 앱은 HSKK 시험 유형과 개인 진도에 맞지 않는다.

그래서 목표를 이렇게 잡았다.

“내가 원할 때, 모바일에서 바로 열고, 오늘 해야 할 HSKK 말하기 연습만 시작하게 만들자.”

현재는 Codex와 Harness를 활용해 매일 HSKK 레슨을 만들고, GitHub Pages에 모바일 페이지로 배포하며, AI Speaking Trainer에서 녹음, STT, AI 평가, 원어민 스타일 발음 비교, 교정 답변 발음 비교까지 이어지도록 발전했다.

3. 5장 발표자료 구성안
============================================================

------------------------------------------------------------
Slide 1. 왜 공부해도 말하기는 늘지 않을까?
------------------------------------------------------------

슬라이드 목적:
청중의 공감을 끌어낸다. 기술 이야기를 시작하기 전에, 언어학습의 실제 문제를 보여준다.

제목:
왜 공부해도 말하기는 늘지 않을까?

핵심 문장:
HSKK는 말하기 시험인데, 실제로 말하고 피드백받는 시간은 가장 부족하다.

주요 내용:
기존 학습은 단어 암기, 문법 학습, 문제 풀이, 정답 확인으로 끝나는 경우가 많다.
하지만 말하기 학습에는 다른 문제가 있다.

- 배운 표현을 실제로 말할 기회가 부족하다.
- 혼자 공부하면 답변이 맞는지 바로 알기 어렵다.
- 피드백을 받아도 다음 학습에 체계적으로 연결되지 않는다.
- 범용 회화 앱은 HSKK 시험 유형과 개인 진도에 맞지 않는다.

하단 강조 문구:
공부한 내용을 실제 말하기로 연결하고, 부족한 부분을 다음 학습에 다시 반영하는 구조가 필요했다.

시각화 제안:
- 왼쪽: 책, 단어, 문법, 문제풀이 아이콘
- 오른쪽: 마이크, 피드백, 반복 학습 아이콘
- 중앙: “학습”과 “말하기” 사이가 끊어진 모습
- 끊어진 다리 또는 비어 있는 말풍선으로 표현

발표자 노트:
이 프로젝트는 공부를 더 열심히 하기 위한 도구가 아니라, 공부를 실제 말하기로 연결하기 위한 도구입니다. HSKK는 말하기 시험인데, 혼자 공부하면 실제로 말하는 시간이 가장 부족합니다. 그래서 학습 콘텐츠, 녹음, 피드백, 복습이 하나로 이어지는 구조가 필요했습니다.

------------------------------------------------------------
Slide 2. AI가 학습 전체를 운영한다
------------------------------------------------------------

슬라이드 목적:
이 프로젝트가 단순 AI 회화 앱이 아니라 Learning Loop라는 점을 보여준다.

제목:
AI가 학습 전체를 운영한다

핵심 구조:
오늘 학습 → AI와 말하기 → AI 평가 → 약점 분석 → Harness 반영 → 다음 학습 개선

강조 문구:
한 번의 대화가 아니라, 매일 발전하는 학습 구조

주요 설명:
일반 AI 챗봇은 대화가 끝나면 학습도 끝난다.
이 프로젝트는 사용자의 답변과 오류를 저장하고, 반복되는 약점을 분석한 뒤, 다음 레슨의 콘텐츠와 복습 항목을 바꾸는 구조를 지향한다.

현재 구현 기준:
- 매일 레슨 생성과 모바일 페이지 배포는 자동화됨.
- 녹음, STT, AI 평가, TTS 발음 비교는 Trainer에서 동작함.
- 학습 결과는 JSON export와 local History로 남길 수 있음.
- 다음 학습 자동 반영은 현재 완전 자동이 아니라 JSON export 또는 Codex 입력 기반 반자동 루프임.

AI 역할을 한글 일꾼으로 표현:
- 콘텐츠 제작자: 오늘의 레슨과 질문 준비
- 말하기 파트너: 사용자가 실제로 말하도록 유도
- 시험관: HSKK 스타일 답변 평가
- 교정 코치: 더 자연스러운 답변 제안
- 발음 코치: 모범 발음과 교정 답변 발음 제공
- 복습 관리자: 약점과 다음 연습 추천 정리
- 배포 관리자: 모바일 페이지 갱신과 GitHub Pages 반영

시각화 제안:
- 중앙에 학습자 또는 스마트폰 화면
- 주변에 6~7명의 AI 일꾼
- 원형 화살표로 Learning Loop 표시
- 영어 Agent 명칭보다 한글 역할명을 사용

발표자 노트:
핵심은 AI가 한 번 답을 만들어 주는 것이 아닙니다. 콘텐츠를 만들고, 사용자가 말하게 하고, 평가하고, 다시 연습할 포인트를 남기는 운영 구조입니다. 저는 이 구조를 Harness 기반 Learning Loop로 만들었습니다.

------------------------------------------------------------
Slide 3. 공부에서 실제 말하기까지, 한 화면에서
------------------------------------------------------------

슬라이드 목적:
사용자 경험을 보여준다. 청중이 실제 사용 흐름을 한 번에 이해하도록 만든다.

제목:
공부에서 실제 말하기까지, 한 화면에서

사용자 흐름:
1. 홈 화면 바로가기 실행
2. 오늘의 학습자료 확인
3. 한자, 병음, 한국어 뜻 확인
4. 자체 피드백 메모 작성
5. “녹음하고 AI 평가 받기”로 Trainer 이동
6. 문제를 보면서 답변 녹음
7. 내 녹음 재생
8. AI 평가 확인
9. 원어민 스타일 발음과 교정 답변 발음 비교
10. 다시 말하기 또는 레슨 계속 보기

핵심 기능:
- 오늘 학습한 표현 기반 AI 말하기 연습
- 실제 음성 녹음
- 내 녹음 재생
- 문법, 어휘, 유창성, 발음, 과제 완성도 피드백
- 더 자연스러운 교정 문장 제공
- 교정 문장으로 다시 말하기
- HSKK 예상문제 Test
- 레슨 기반 예상문제 자동 표시
- JSON export와 History 저장

핵심 메시지:
읽고 끝나는 학습이 아니라, 말하고, 평가받고, 다시 말하는 학습

시각화 제안:
모바일 화면 3개를 순서대로 배치한다.

화면 1:
오늘의 HSKK 레슨
한자 + 병음 + 한국어 뜻

화면 2:
HSKK AI Trainer
녹음 중 또는 내 녹음 들어보기

화면 3:
AI 코칭 리포트
교정 답변 + 원어민 발음 비교

발표자 노트:
사용자는 GitHub Pages를 홈 화면 바로가기로 열고 오늘 레슨을 봅니다. 레슨 하단에서 Trainer로 이동하면 같은 레슨 내용을 보면서 녹음할 수 있습니다. 녹음 후에는 내 녹음을 먼저 들어보고, AI 평가를 실행합니다. 결과에서는 강점, 약점, 교정 답변을 확인하고 모범 발음까지 들을 수 있습니다.

------------------------------------------------------------
Slide 4. 기존 유료 앱과 무엇이 다른가?
------------------------------------------------------------

슬라이드 목적:
차별점을 명확히 한다. “그냥 스픽 같은 앱 쓰면 되는 것 아닌가?”라는 질문에 답한다.

제목:
단순 AI 회화 앱이 아니라, 개인 맞춤형 학습 운영 시스템

비교표:

기존 유료 AI 회화 앱:
- AI와 대화
- 피드백 후 종료
- 범용 회화 중심
- 앱이 정한 학습 방식
- 데이터 활용 제한
- 학습 로직 수정 불가

HSKK AI Speaking Trainer:
- AI가 학습 전체 운영
- 피드백이 다음 학습으로 연결 가능한 구조
- HSKK 시험 유형 특화
- 내 목적에 맞게 직접 설계
- 오류와 학습 데이터 누적
- Codex와 Harness로 계속 개선 가능

핵심 차별점:
일반 앱:
대화 → 피드백 → 종료

내 프로젝트:
학습 → 말하기 → 평가 → 약점 저장 → 다음 학습 반영 → 반복

강조 문구:
AI가 사용자를 기억하고, 다음 학습을 계속 개선한다.

주의 표현:
현재 “자동으로 기억하고 반영”이 완전히 구현된 것은 아니다.
발표에서는 “JSON export와 Harness import를 염두에 둔 구조”, “반자동 루프”, “확장 가능한 구조”라고 표현한다.

시각화 제안:
- 왼쪽: 직선형 프로세스
- 오른쪽: 원형 Learning Loop
- 왼쪽은 앱 화면 하나와 종료 아이콘
- 오른쪽은 레슨, 녹음, AI 피드백, JSON, 복습 큐가 순환하는 구조

발표자 노트:
유료 AI 회화 앱은 훌륭하지만 보통 앱이 정한 흐름을 따라갑니다. 이 프로젝트는 제가 필요한 HSKK 유형, 제가 만든 레슨, 제 약점, 제 복습 구조를 직접 바꿀 수 있다는 점이 다릅니다. 핵심은 대화가 끝나도 학습이 끝나지 않는다는 점입니다.

------------------------------------------------------------
Slide 5. 비용은 낮추고, 개인화와 확장성은 높였다
------------------------------------------------------------

슬라이드 목적:
AX 관점에서 실용성과 확장성을 보여준다. 단순 취미 프로젝트가 아니라 비용 대비 가치와 확장 가능한 운영 모델임을 강조한다.

제목:
비용은 낮추고, 개인화와 확장성은 높였다

비용 비교:
- 자체 프로젝트 예상 운영비: 월 약 1,500원~4,500원
- 유사 유료 AI 회화 앱: 월 약 15,000원~30,000원

비용 메시지:
월 운영비 기준으로 자체 구축이 약 70~90% 저렴할 가능성이 있다.

중요 주석:
자체 프로젝트 비용은 사용량, 모델, 음성 길이, STT/TTS 호출량에 따라 달라질 수 있다.
유료 앱 가격도 플랜, 국가, 프로모션에 따라 달라질 수 있다.
비용 비교는 고정 가격 보장이 아니라 개인 사용량 기준의 추정 비교다.

더 중요한 가치:
단순히 비용이 저렴한 것이 핵심은 아니다.

- HSKK에 맞게 기능을 직접 설계할 수 있다.
- 내 학습 데이터를 직접 관리할 수 있다.
- 반복 오류를 다음 학습에 반영할 수 있다.
- 필요하면 다른 시험과 업무로 확장할 수 있다.
- 특정 앱의 학습 방식에 갇히지 않는다.

확장 가능 분야:
HSKK → OPIC → 영어 스피킹 → 면접 연습 → 사내교육 → 업무 AX

마지막 강조 문구:
AI가 공부를 대신하는 것이 아니라,
AI가 학습을 운영한다.

보조 문구:
HSKK 학습에서 시작했지만, 이 구조는 반복 업무를 데이터 기반으로 개선하는 AI Agent 운영 모델로 확장할 수 있다.

시각화 제안:
- 상단: 비용 비교 숫자 크게 표시
- 하단: HSKK에서 OPIC, 면접, 사내교육, 업무 AX로 확장되는 화살표
- 마지막 문장은 큰 타이포그래피로 마무리

발표자 노트:
비용만 보면 API 기반 자체 구축이 개인 사용량에서는 훨씬 저렴할 가능성이 있습니다. 하지만 더 중요한 점은 비용보다 통제권입니다. 내가 원하는 시험 유형, 내가 원하는 학습 방식, 내가 쌓은 데이터를 바탕으로 계속 바꿔 갈 수 있습니다. HSKK에서 시작했지만, 이 구조는 사내교육이나 업무 AX에도 그대로 확장할 수 있습니다.

5. 발표자 노트용 기술 설명
============================================================

이 내용은 본문 슬라이드에 자세히 넣지 말고, 질문이 나왔을 때 설명한다.

현재 시스템 구조:
사용자 모바일 브라우저
→ GitHub Pages 정적 UI
→ Vercel Serverless API
→ OpenAI API
→ STT, AI 평가, TTS
→ 브라우저 History와 JSON Export
→ 향후 Harness import

구성요소 역할:
- GitHub Pages: 학습 화면과 콘텐츠 제공
- 모바일 레슨 JSON: 오늘 레슨과 Trainer 연결
- Vercel API: OpenAI API Key 보호 및 API 중계
- OpenAI API: 음성 인식, 답변 평가, 음성 생성
- Browser localStorage: 세션 History 저장
- JSON export: Harness 연계용 학습 결과 저장
- Harness: 레슨 생성, 복습 큐, 모바일 페이지 갱신, daily loop 운영

왜 Vercel API를 사용하는가?
초기에는 Cloudflare Worker를 사용했지만 OpenAI STT 호출에서 지역 제한 오류가 발생했다.
Smart Placement도 시도했으나 안정적으로 해결되지 않아, OpenAI API 호출 위치를 더 안정적으로 고정할 수 있는 Vercel Serverless Function으로 이전했다.

보안:
- 브라우저에는 OpenAI API Key를 넣지 않는다.
- OpenAI API Key는 Vercel Environment Variable에만 저장한다.
- GitHub Pages는 화면만 담당한다.
- Vercel API가 OpenAI 호출을 담당한다.

비용:
- 녹음 자체는 로컬 처리라 비용이 없다.
- 내 녹음 재생도 비용이 없다.
- STT, AI 평가, TTS를 실행할 때만 API 비용이 발생한다.
- 그래서 사용자가 “평가하기”를 누를 때만 API를 호출한다.

AI 평가의 한계:
- 공식 HSKK 점수가 아니다.
- 연습용 예상 평가와 코칭 피드백이다.
- 발음 평가는 음성 인식 결과와 답변 품질 기반의 보조 평가이며, 정밀 성조 채점은 향후 고도화 항목이다.

7. 예상 질문과 답변
============================================================

Q1. 왜 직접 만들었는가?
기존 유료 앱은 범용 회화에 강하지만 HSKK 유형, 개인 복습 구조, 학습 데이터 반영에는 한계가 있다. 이 프로젝트는 내 학습 목적에 맞게 콘텐츠, 평가, 복습 로직을 직접 설계할 수 있다.

Q2. AI 평가를 믿을 수 있는가?
공식 성적을 대체하는 용도가 아니다. 문법, 어휘, 답변 구조, 유창성, 반복 오류를 분석하고 개선 방향을 제시하는 학습 보조 평가다.

Q3. 비용이 왜 저렴한가?
사용한 만큼만 API 비용을 지불하고, GitHub Pages와 Vercel 같은 저비용 배포 구조를 활용하기 때문이다. 단, 사용량과 모델에 따라 달라질 수 있다.

Q4. 일반 챗봇과 무엇이 다른가?
대화가 끝나지 않고 학습 결과가 다음 레슨에 다시 반영될 수 있는 구조를 지향한다는 점이 다르다. 일반 챗봇이 대화 도구라면, 이 프로젝트는 학습 운영 루프다.

Q5. 자기채점 메모가 자동으로 다음 레슨에 반영되나?
현재는 완전 자동이 아니다. 사용자가 JSON export를 하거나 Codex에 피드백을 전달하면 다음 레슨과 복습 계획에 반영할 수 있다. 완전 자동 반영은 향후 서버 저장소나 Harness import 자동화가 필요하다.

Q6. 다른 분야에도 적용 가능한가?
가능하다. 콘텐츠 제공 → 사용자 실행 → 결과 수집 → 평가 → 개선 → 다음 작업 반영 구조는 교육뿐 아니라 면접, 직무교육, 사무 자동화, 품질 관리에도 활용할 수 있다.

Q7. Cloudflare Worker가 아니라 Vercel API를 쓰는 이유는?
Cloudflare Worker에서 OpenAI STT 지역 제한 오류가 발생해, 운영 백엔드는 Vercel API로 전환했다. GitHub Pages는 화면을 담당하고, Vercel은 OpenAI API 호출을 담당한다.

Q8. HSKK 예상문제 Test는 상용 문제은행인가?
아니다. 저작권 있는 상용 문제은행이 아니라, 자체 생성 레슨과 HSKK 형식 기반의 연습 문제다.

Q9. 원어민 발음 비교는 정확히 무엇인가?
내 녹음이 STT로 transcript가 되면, 그 transcript를 OpenAI TTS로 다시 생성해 모범 발음처럼 들려준다. 평가 후에는 AI가 다듬은 교정 답변도 TTS로 들을 수 있다.
