1. 핵심 메시지 관련 질문
============================================================

Q1. 이 프로젝트를 한 문장으로 설명하면 무엇인가요?

A.
HSKK AI Trainer는 스마트폰에서 오늘의 중국어 말하기 레슨을 바로 열고, 녹음하고, AI 피드백과 모범 발음을 비교하며 다시 연습하는 개인 맞춤형 학습 운영 루프입니다.

짧은 답변:
공부를 대신해 주는 앱이 아니라, 매일 말하기 학습을 준비하고 피드백까지 연결해 주는 AI 학습 루프입니다.

------------------------------------------------------------

Q2. 왜 “의지 부족”이 아니라 “시작까지의 마찰력”이 문제라고 보나요?

A.
언어학습을 하려는 마음이 있어도, 실제로 시작하려면 자료를 찾고, 진도를 확인하고, 앱을 실행하고, 어떤 문제를 풀지 정해야 합니다. 이 과정이 길어지면 실제 말하기 전에 동기가 떨어집니다. 그래서 이 프로젝트는 홈 화면에서 바로 들어가 오늘 해야 할 HSKK 학습을 시작하게 만드는 데 초점을 맞췄습니다.

짧은 답변:
공부 의지가 없는 것이 아니라, 시작 전 준비 과정이 길어서 실제 말하기까지 못 가는 문제가 컸습니다.

------------------------------------------------------------

Q3. 기존 발표자료의 “왜 공부해도 말하기는 늘지 않을까?”와 지금 메시지는 어떻게 다른가요?

A.
기존 메시지는 말하기 학습의 일반적인 문제를 설명했다면, 이번 자료는 제가 왜 이 프로젝트를 만들었는지에 더 집중합니다. 핵심은 “말하기가 안 느는 이유”보다 “내가 원할 때 바로 시작할 수 있는 개인 학습 루틴이 필요했다”는 개인적 문제의식입니다.

짧은 답변:
이번 자료는 문제 분석보다 제작 동기와 사용 경험을 먼저 보여주는 방향입니다.

============================================================
2. 기능과 사용자 흐름 관련 질문
============================================================

Q4. 사용자는 실제로 어떤 순서로 학습하나요?

A.
사용자는 스마트폰 홈 화면의 HSKK 바로가기를 누르고 오늘 레슨을 확인합니다. 한자, 병음, 한국어 뜻을 보고 레슨 하단에서 Trainer로 이동합니다. 이후 레슨 내용을 보면서 녹음하거나, HSKK 예상문제 Test를 선택해 문제를 풀듯이 녹음합니다. 녹음 후에는 내 녹음을 듣고, AI 평가와 교정 답변, 모범 발음을 확인합니다.

짧은 답변:
홈 화면에서 시작해 레슨 확인, 녹음, 내 녹음 재생, AI 평가, 모범 발음 비교까지 이어집니다.

------------------------------------------------------------

Q5. 녹음 기능이 두 가지라고 했는데 정확히 무엇인가요?

A.
첫 번째는 오늘 레슨 내용을 보면서 녹음하고 AI 평가를 받는 흐름입니다. 레슨에서 배운 한자, 병음, 뜻을 보고 바로 말하기 연습을 합니다.

두 번째는 HSKK 예상문제 Test입니다. Beginner, Intermediate, Advanced 레벨을 선택하고 HSKK 스타일 질문을 보면서 답변을 녹음합니다.

짧은 답변:
하나는 레슨 기반 녹음 평가이고, 다른 하나는 HSKK 예상문제 풀이형 녹음 평가입니다.

------------------------------------------------------------

Q6. 내 녹음을 다시 들을 수 있는 이유가 왜 중요한가요?

A.
말하기 학습에서는 AI 평가만큼 자기 인식도 중요합니다. 사용자가 먼저 자신의 발음, 속도, 멈춤, 문장 구성을 직접 들어본 뒤 평가를 실행하면 피드백을 더 잘 이해할 수 있습니다. 그래서 녹음 후 바로 내 녹음을 재생하는 흐름을 넣었습니다.

짧은 답변:
내 목소리를 먼저 들어야 AI 피드백이 무엇을 고치라는 말인지 더 분명해집니다.

------------------------------------------------------------

Q7. 원어민 스타일 발음 비교는 실제 원어민 녹음인가요?

A.
아닙니다. 실제 사람 원어민 녹음은 아니고 OpenAI TTS 기반 모범 발음입니다. 사용자는 내 녹음, STT 결과 기반 모범 발음, AI가 다듬은 교정 답변 발음을 비교할 수 있습니다.

짧은 답변:
실제 원어민 녹음이 아니라 OpenAI TTS 기반 모범 발음입니다.

============================================================
3. AI 평가와 정확도 관련 질문
============================================================

Q8. AI 평가 점수는 공식 HSKK 점수인가요?

A.
아닙니다. 공식 HSKK 점수가 아니라 연습용 AI 예상 평가입니다. 문법, 어휘, 답변 구조, 유창성, 발음 명료도, 과제 완성도 관점에서 개선 방향을 제시하는 코칭 지표로 봐야 합니다.

짧은 답변:
공식 점수가 아니라 연습용 AI 피드백입니다.

------------------------------------------------------------

Q9. AI 평가를 어느 정도 믿을 수 있나요?

A.
공식 시험 채점처럼 절대적인 점수로 보면 안 됩니다. 다만 혼자 공부할 때 놓치기 쉬운 답변 구조, 문법 오류, 어휘 선택, 답변의 구체성, 반복 약점을 빠르게 확인하는 데 유용합니다. 이 프로젝트의 목적은 정확한 성적 예측보다 매일 다시 말할 포인트를 찾는 것입니다.

짧은 답변:
성적 예측보다 개선 포인트를 찾는 코칭 도구로 보는 것이 맞습니다.

------------------------------------------------------------

Q10. 발음 평가도 정확하게 되나요?

A.
현재 발음 평가는 정밀한 성조 채점이라기보다 STT 결과와 답변 품질을 바탕으로 한 보조 평가입니다. 정교한 성조 분석이나 음소 단위 피드백은 향후 고도화 과제입니다.

짧은 답변:
현재는 발음 명료도 중심의 보조 평가이고, 정밀 성조 채점은 향후 과제입니다.

============================================================
4. 기존 유료 앱과 차별점 관련 질문
============================================================

Q11. 그냥 유료 AI 회화 앱을 쓰면 되는 것 아닌가요?

A.
유료 AI 회화 앱은 범용 회화 연습에는 좋습니다. 하지만 HSKK 시험 유형, 개인 레슨 생성, 복습 큐, JSON export, Harness 기반 개선 구조처럼 제가 원하는 방식으로 학습 루프를 바꾸기는 어렵습니다. 이 프로젝트는 앱이 정한 학습 방식이 아니라 내 목적에 맞게 학습 구조를 직접 설계할 수 있다는 점이 다릅니다.

짧은 답변:
유료 앱은 대화 중심이고, 이 프로젝트는 HSKK에 맞춘 개인 학습 운영 시스템입니다.

------------------------------------------------------------

Q12. 발표자료에서 말하는 “학습 운영 시스템”은 무슨 뜻인가요?

A.
단순히 AI와 대화하고 끝나는 것이 아니라, 오늘 레슨을 만들고, 모바일에서 실행하고, 녹음 결과를 평가하고, 약점을 정리하고, 다음 학습 개선으로 연결하는 전체 흐름을 말합니다.

짧은 답변:
레슨 생성부터 평가와 다음 학습 개선까지 이어지는 반복 구조입니다.

------------------------------------------------------------

Q13. “AI가 사용자를 기억한다”는 표현은 정확한가요?

A.
현재 완전 자동으로 모든 학습 데이터를 기억하고 다음 레슨에 반영하는 상태는 아닙니다. 더 정확히는 JSON export와 Codex 입력을 통해 학습 결과를 다음 개선에 반영할 수 있는 구조를 갖췄다는 의미입니다. 발표에서는 “반자동 학습 루프” 또는 “다음 학습으로 연결 가능한 구조”라고 표현하는 것이 안전합니다.

짧은 답변:
완전 자동 기억은 아니고, JSON export와 Codex 입력 기반으로 다음 학습에 반영 가능한 구조입니다.

============================================================
5. 비용과 운영 관련 질문
============================================================

Q14. 자체 프로젝트 운영비가 왜 낮게 나오나요?

A.
녹음 자체와 내 녹음 재생은 브라우저에서 로컬로 처리되어 비용이 없습니다. 비용은 STT, AI 평가, TTS를 실행할 때만 발생합니다. 또한 화면은 GitHub Pages로 제공하고, API는 Vercel Serverless를 사용해 개인 사용량 기준으로 낮은 운영비를 기대할 수 있습니다.

짧은 답변:
필요할 때만 API를 호출하고, 나머지는 GitHub Pages와 브라우저 로컬 기능을 활용하기 때문입니다.

------------------------------------------------------------

Q15. 월 1,500원~4,500원 비용은 확정인가요?

A.
아닙니다. 개인 사용량 기준의 추정치입니다. 실제 비용은 녹음 길이, 평가 횟수, STT/TTS 호출량, 사용하는 모델, 환율에 따라 달라질 수 있습니다. 유료 앱 가격도 플랜과 프로모션에 따라 다를 수 있습니다.

짧은 답변:
확정 가격이 아니라 개인 사용량 기준 추정치입니다.

------------------------------------------------------------

Q16. 보안상 API Key가 노출될 위험은 없나요?

A.
브라우저에는 OpenAI API Key를 넣지 않습니다. GitHub Pages는 정적 화면만 제공하고, OpenAI 호출은 Vercel Serverless API가 담당합니다. API Key는 Vercel Environment Variable에만 저장됩니다.

짧은 답변:
API Key는 브라우저가 아니라 Vercel 서버 환경변수에만 저장합니다.

------------------------------------------------------------

Q17. 왜 Cloudflare Worker가 아니라 Vercel API를 쓰나요?

A.
초기에는 Cloudflare Worker를 사용했지만 OpenAI STT 호출에서 지역 제한 오류가 발생했습니다. Smart Placement도 시도했지만 안정적으로 해결되지 않아 Vercel Serverless API로 이전했습니다. 현재 운영 기준은 GitHub Pages + Vercel API + OpenAI API입니다.

짧은 답변:
Cloudflare Worker에서 STT 지역 제한 오류가 있어 Vercel API로 전환했습니다.

6. 확장성 관련 질문
============================================================

Q18. HSKK 외 다른 학습에도 쓸 수 있나요?

A.
가능합니다. 구조 자체는 특정 언어에만 묶여 있지 않습니다. 콘텐츠 제공, 사용자 실행, 결과 수집, AI 평가, 다음 작업 개선이라는 흐름이기 때문에 OPIC, 영어 스피킹, 면접 연습, 사내교육으로 확장할 수 있습니다.

짧은 답변:
HSKK에서 시작했지만 OPIC, 영어 스피킹, 면접, 사내교육으로 확장 가능합니다.

------------------------------------------------------------

Q19. 업무 AX로 확장한다는 말은 어떤 의미인가요?

A.
반복 업무에서도 비슷한 구조가 가능합니다. 작업 지침을 제공하고, 사용자가 실행하고, 결과를 수집하고, AI가 평가한 뒤 다음 작업 방식을 개선하는 구조입니다. 교육뿐 아니라 품질 점검, 직무교육, 보고서 작성, 고객 응대 훈련 같은 영역에도 적용할 수 있습니다.

짧은 답변:
반복 업무의 실행 결과를 AI가 평가하고 다음 작업을 개선하는 운영 모델로 확장할 수 있다는 뜻입니다.

------------------------------------------------------------

Q20. 앞으로 가장 먼저 고도화할 부분은 무엇인가요?

A.
가장 중요한 고도화는 학습 결과 자동 반영입니다. 현재는 JSON export 또는 Codex 입력 기반 반자동 루프입니다. 다음 단계는 exported JSON이나 피드백 메모를 Harness가 자동으로 가져와 복습 큐와 다음 레슨에 반영하는 구조입니다.

짧은 답변:
다음 단계는 JSON export와 피드백 메모를 Harness에 자동 반영하는 것입니다.
