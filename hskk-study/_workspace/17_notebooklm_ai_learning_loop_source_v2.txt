1. 프로젝트 핵심 배경
============================================================

이 프로젝트의 출발점은 거창한 플랫폼 개발이 아니라 아주 개인적인 불편이었다.

중국어 HSKK 말하기를 준비하려면 매일 실제로 말해야 한다.
하지만 혼자 공부할 때는 말하기 자체보다 시작 전 과정이 더 번거로웠다.

문제는 의지 부족이 아니었다.
문제는 시작까지의 마찰력이었다.

학습자가 겪는 마찰:
- 오늘 무엇을 공부할지 고른다.
- 자료를 찾는다.
- 진도를 확인한다.
- 앱을 실행한다.
- 로딩을 기다린다.
- HSKK 유형에 맞는 질문과 답변을 찾는다.
- 한자, 병음, 한국어 뜻을 정리한다.
- 녹음하고 다시 들어볼 도구를 찾는다.
- 피드백을 따로 정리한다.

이 과정이 길어지면 학습 동기는 실제 말하기 전에 이미 꺼진다.

그래서 이 프로젝트의 목표는 단순했다.

“홈 화면에서 바로 열고, 오늘 해야 할 HSKK 말하기 연습만 시작하자.”

지향하는 사용자 경험:
- 홈 화면 터치 1초.
- 오늘의 HSKK 레슨 즉시 확인.
- 한자, 병음, 뜻을 보고 바로 말하기.
- 녹음하고 내 목소리 확인.
- AI 피드백과 모범 발음으로 다시 연습.
- 결과를 저장하고 다음 학습으로 연결.

------------------------------------------------------------
Page 1. 내가 왜 만들었는가
------------------------------------------------------------

슬라이드 역할:
청중의 공감을 먼저 만든다. “말하기가 늘지 않는 이유”보다 “왜 이 개인 학습 루틴을 만들었는지”를 전면에 둔다.

제목 후보:
내가 원할 때 나만의 컨텐츠로

부제 후보:
언어 학습의 실패는 의지 부족이 아니라, 시작까지의 마찰력 때문입니다.

핵심 카피:
자료 검색 → 진도 확인 → 앱 실행 → 로딩 대기
이 과정에서 이미 학습 동기가 꺼집니다.

반대편 카피:
홈 화면 터치 1초 → 즉시 학습 시작
오늘 해야 할 HSKK 말하기만 바로 시작합니다.

하단 강조 문구:
공부할 결심보다 쉬운 중국어.
스마트폰에서 열고, 말하고, 바로 피드백 받는 AI 학습 루틴.

시각화 방향:
- 첨부 이미지 1과 2의 메시지를 섞는다.
- 왼쪽은 “마찰이 많은 기존 학습”을 표현한다.
  - 책, 자료, 여러 앱, 로딩 화면, 스프레드시트, 진도표.
  - 흐름: 자료 검색 → 진도 확인 → 앱 실행 → 로딩 대기.
- 오른쪽은 “즉시 시작하는 모바일 학습”을 표현한다.
  - 밝은 휴대폰 이미지.
  - 홈 화면 위 HSKK 바로가기.
  - 손가락이 아이콘을 터치하는 장면.
  - 말풍선: 원할 때 바로 시작, 모바일 바로가기, 매일 30분, AI 학습 루틴.
- 전체 톤은 흰 배경, 연한 블루, 부드러운 유리 느낌, 밝은 그림자.

발표자 노트:
이 프로젝트는 “공부를 더 열심히 해야겠다”에서 출발한 것이 아닙니다. 오히려 반대였습니다. 언어학습을 하고 싶은 순간은 있는데, 앱을 찾고 자료를 고르고 진도를 정리하는 과정이 너무 번거로웠습니다. 그래서 목표를 단순하게 잡았습니다. 홈 화면에서 바로 열고, 오늘 해야 할 HSKK 말하기만 시작하자. 언어 학습의 실패는 의지 부족이 아니라 시작까지의 마찰력 때문이라고 생각했습니다.

------------------------------------------------------------
Page 2. AI가 학습 전체를 운영한다
------------------------------------------------------------

슬라이드 역할:
Learning Loop를 밝고 쉬운 톤으로 설명한다. 기술 구조가 아니라 “학습이 어떻게 매일 돌아가는지”를 보여준다.

제목:
AI가 학습 전체를 운영한다

핵심 문장:
한 번의 대화가 아니라, 매일 발전하는 학습 루프.

원형 Loop:
1. 오늘 학습 준비
2. 모바일에서 확인
3. 말하기 연습
4. AI 피드백
5. 약점 정리
6. 다음 학습 개선

AI 일꾼 역할:
- 콘텐츠 제작자: 오늘의 HSKK 레슨과 질문 준비
- 말하기 파트너: 사용자가 실제로 말하도록 유도
- 시험관: HSKK 스타일 답변 평가
- 교정 코치: 더 자연스러운 답변 제안
- 발음 코치: 모범 발음과 교정 답변 발음 제공
- 복습 관리자: 약점과 다음 연습 추천 정리
- 배포 관리자: 모바일 페이지 갱신

시각화 방향:
- 중앙에 밝은 스마트폰 또는 학습자를 둔다.
- 주변에 친근한 AI 일꾼들이 둘러싸고 있다.
- 일꾼들은 작은 작업 도구를 들고 있다.
  - 문서, 마이크, 체크리스트, 이어폰, 달력, 업로드 아이콘.
- 원형 화살표는 밝은 블루 또는 민트 색상.
- 복잡한 시스템 다이어그램처럼 보이지 않게 한다.

주의 문구:
현재 완전 자동 반영이 아니라, JSON export와 Codex 입력을 통한 반자동 루프입니다.

발표자 노트:
이 프로젝트의 핵심은 AI가 한 번 답을 주는 것이 아닙니다. AI가 오늘의 학습을 준비하고, 제가 말하게 만들고, 피드백을 주고, 다시 연습할 포인트를 남깁니다. 즉 단발성 챗봇이 아니라 매일 반복되는 학습 운영 체계입니다.

제목:
공부에서 실제 말하기까지, 한 화면에서

핵심 메시지:

- 문제를 보면서 녹음

- 내 녹음 들어보기
- 다시 녹음
- AI 평가

- 두 갈래는 복잡한 플로우차트가 아니라 카드 2개로 간단히 보여준다.

발표자 노트:
Trainer의 녹음 흐름은 두 가지입니다. 첫 번째는 오늘 레슨을 보고 바로 녹음해서 AI 평가를 받는 방식입니다. 두 번째는 HSKK 예상문제 Test에서 문제를 풀듯이 녹음하는 방식입니다. 중요한 점은 읽고 끝나는 학습이 아니라, 실제로 말하고, 내 녹음을 듣고, AI 피드백으로 다시 말하는 구조라는 점입니다.

============================================================
4. 예상 질문과 답변
============================================================

Q1. 왜 직접 만들었나?
기존 유료 앱은 범용 회화에 강하지만 HSKK 유형, 개인 복습 구조, 학습 데이터 반영에는 한계가 있다. 이 프로젝트는 내 학습 목적에 맞게 콘텐츠, 평가, 복습 로직을 직접 설계할 수 있다.

Q2. AI 평가를 믿을 수 있나?
공식 성적을 대체하는 용도가 아니다. 문법, 어휘, 답변 구조, 유창성, 반복 오류를 분석하고 개선 방향을 제시하는 연습용 AI 피드백이다.

Q3. 비용이 왜 저렴한가?
녹음 자체는 로컬에서 처리하고, STT/AI 평가/TTS를 실행할 때만 API 비용이 발생한다. GitHub Pages와 Vercel 기반 저비용 구조를 활용한다. 단, 실제 비용은 사용량과 모델에 따라 달라질 수 있다.

Q4. 일반 챗봇과 무엇이 다른가?
일반 챗봇은 대화가 끝나면 학습도 끝나는 경우가 많다. 이 프로젝트는 레슨, 녹음, 평가, 복습, 다음 학습 개선으로 이어지는 Learning Loop를 지향한다.

Q5. 자기채점 메모가 자동으로 다음 레슨에 반영되나?
현재는 완전 자동이 아니다. 사용자가 JSON export를 하거나 Codex에 피드백을 전달하면 다음 레슨과 복습 계획에 반영할 수 있다. 완전 자동 반영은 향후 서버 저장소나 Harness import 자동화가 필요하다.

Q6. HSKK 예상문제 Test는 상용 문제은행인가?
아니다. 저작권 있는 상용 문제은행이 아니라, 자체 생성 레슨과 HSKK 형식 기반의 연습 문제다.

Q7. 원어민 발음 비교는 실제 원어민 음성인가?
실제 사람 원어민 녹음은 아니다. OpenAI TTS 기반 모범 발음이다. 사용자는 내 녹음, STT transcript 기반 모범 발음, AI 교정 답변 기반 모범 발음을 비교할 수 있다.

Q8. Cloudflare Worker가 아니라 Vercel API를 쓰는 이유는?
Cloudflare Worker에서 OpenAI STT 지역 제한 오류가 발생해, 운영 백엔드는 Vercel API로 전환했다. GitHub Pages는 화면을 담당하고, Vercel은 OpenAI API 호출을 담당한다.

Q9. 다른 분야에도 적용 가능한가?
가능하다. 콘텐츠 제공 → 사용자 실행 → 결과 수집 → 평가 → 개선 → 다음 작업 반영 구조는 교육뿐 아니라 면접, 직무교육, 사내교육, 품질 관리, 업무 AX에도 활용할 수 있다.

6. 기술 설명은 발표자 노트에만 사용
============================================================

현재 시스템 구조:
사용자 모바일 브라우저
→ GitHub Pages 정적 UI
→ Vercel Serverless API
→ OpenAI API
→ STT, AI 평가, TTS
→ 브라우저 History와 JSON Export
→ 향후 Harness import

구성요소 역할:
- GitHub Pages: 학습 화면과 콘텐츠 제공
- 모바일 레슨 JSON: 오늘 레슨과 Trainer 연결
- Vercel API: OpenAI API Key 보호 및 API 중계
- OpenAI API: 음성 인식, 답변 평가, 음성 생성
- Browser localStorage: 세션 History 저장
- JSON export: Harness 연계용 학습 결과 저장
- Harness: 레슨 생성, 복습 큐, 모바일 페이지 갱신, daily loop 운영

보안:
- 브라우저에는 OpenAI API Key를 넣지 않는다.
- OpenAI API Key는 Vercel Environment Variable에만 저장한다.
- GitHub Pages는 화면만 담당한다.
- Vercel API가 OpenAI 호출을 담당한다.

비용:
- 녹음 자체는 로컬 처리라 비용이 없다.
- 내 녹음 재생도 비용이 없다.
- STT, AI 평가, TTS를 실행할 때만 API 비용이 발생한다.
- 그래서 사용자가 “평가하기”를 누를 때만 API를 호출한다.

7. 최종 발표 내러티브
============================================================

Opening:
언어학습에서 가장 어려운 것은 공부할 마음보다 시작까지의 마찰입니다. 오늘 무엇을 할지 고르고, 자료를 찾고, 진도를 확인하고, 앱을 실행하는 과정이 반복되면 학습은 실제 말하기 전에 끊깁니다.

Personal reason:
저는 중국어 HSKK 말하기를 준비하면서, 언제든 휴대폰에서 바로 열 수 있고, 오늘 필요한 내용만 바로 말할 수 있는 개인 학습 루틴이 필요했습니다.

Solution:
그래서 Codex와 Harness를 이용해 매일 학습 콘텐츠를 만들고, GitHub Pages에 모바일 화면으로 배포하고, AI Speaking Trainer에서 녹음과 평가까지 연결했습니다.

Demo:
사용자는 홈 화면 바로가기를 누르고 오늘 레슨을 봅니다. 레슨 하단에서 Trainer로 이동해 레슨 내용을 보면서 녹음하거나, HSKK 예상문제 Test에서 문제를 풀듯이 녹음할 수 있습니다. 이후 내 녹음을 듣고, AI 피드백과 모범 발음을 비교합니다.

Difference:
기존 앱은 보통 대화와 피드백 후 종료됩니다. 이 프로젝트는 레슨, 녹음, 평가, 약점 저장, 다음 학습 개선으로 이어지는 Learning Loop를 지향합니다.

Closing:
핵심은 AI가 공부를 대신하는 것이 아닙니다. AI가 학습 준비와 피드백 구조를 운영하고, 사람은 실제 말하기 연습에 집중하도록 만드는 것입니다.

마지막 문장:
AI가 공부를 대신하는 것이 아니라,
AI가 학습 루프를 운영합니다.
