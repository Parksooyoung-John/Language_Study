## 발표 핵심 메시지

> 공부할 결심보다 쉬운 중국어. 매일 30분, 내 손안의 AI 학습 루틴.

이 프로젝트는 거창한 플랫폼 개발이 아니라 개인적인 학습 불편에서 시작했습니다.

언어학습을 하고 싶은 순간은 있지만, 자료를 고르고, 오늘 할 일을 정하고, 기록을 정리하고, 다시 복습하는 과정이 번거로웠습니다. 그래서 목표를 단순하게 잡았습니다.

> 내가 원할 때 휴대폰에서 바로 열고, 오늘 해야 할 HSKK 말하기 학습만 집중해서 끝내는 구조를 만들자.

Codex와 Harness는 이 루틴의 뒤에서 매일 학습 콘텐츠를 만들고, 모바일 페이지를 갱신하고, 복습 흐름을 관리하는 역할을 합니다.

핵심 비유:

> 학습자의 목표와 피드백이라는 재료가 들어오면, Agent들이 각 작업대에서 오늘의 학습 콘텐츠를 만들어 모바일로 포장해 준다.

이미지 구성:

- 가운데: 밝고 친근한 작업대 5개
  - 계획 작업대: Curriculum Designer
  - 레슨 작업대: Lesson Tutor
  - 분석 작업대: Error Analyst
  - 복습 작업대: Review Coach
  - 배포 작업대: Automation Runner
- 오른쪽: 완성품
  - 오늘의 레슨 Markdown
  - 모바일 학습 페이지
  - 복습 큐
  - 카카오 알림

이미지 생성 프롬프트:

> A clean friendly infographic for business presentation, A4 landscape composition, showing an AI learning content production line. On the left, baskets labeled learner goal, exam format, previous lessons, self-score notes. In the center, five warm workstation desks with friendly AI agents working: planner, tutor, analyst, review coach, automation runner. On the right, finished outputs: mobile lesson screen, markdown lesson, review calendar, Kakao reminder. Use simple icons, clear arrows, soft colors, Korean labels, no heavy industrial factory, no clutter, easy to understand at first glance.

## 예상 질문과 답변

### Q1. NotebookLM에 요청할 때 PPT 슬라이드 사이즈를 A4 가로로 요청할 수 있나?

답변:

요청 문구에 A4 가로형, 297mm x 210mm를 명시하는 것은 가능합니다. 다만 NotebookLM이 실제 출력 파일의 페이지 크기를 항상 정확히 고정해 준다고 보장하기는 어렵습니다.

권장 방식:
- NotebookLM 프롬프트에 `A4 landscape, 297mm x 210mm`를 명시합니다.
- 여백을 넉넉하게 잡고, 슬라이드 가장자리까지 꽉 채우는 디자인은 피합니다.
- 결과물이 16:9로 나오면 PowerPoint 또는 Google Slides에서 페이지 설정을 A4 가로로 바꾼 뒤 레이아웃을 정리합니다.

발표용 짧은 답변:

> 요청은 가능하지만 보장은 어렵습니다. 그래서 원고에 A4 가로 조건을 넣고, 후처리에서 슬라이드 크기를 A4로 맞추는 방식이 안전합니다.

### Q2. GitHub Pages를 모바일 바로가기로 쓰고 있는데, 범용으로 쓰려면?

답변:

현재 구조는 개인용 앱처럼 쓰는 GitHub Pages입니다. 범용 서비스처럼 만들려면 PWA와 사용자별 데이터 저장 구조가 필요합니다.

추가할 요소:
- web app manifest
- 앱 아이콘
- service worker
- 오프라인 캐시
- 사용자별 진행 상태 저장
- 피드백 제출 폼 또는 데이터베이스

짧은 답변:

> 지금은 개인용 앱처럼 쓰는 웹페이지이고, 범용화하려면 PWA와 사용자별 피드백 저장소가 필요합니다.

### Q3. 자기채점과 피드백이 다음 학습에 자동 반영되나?

답변:

현재는 완전 자동 반영이 아닙니다. 모바일 페이지에서 자기채점 메모를 입력할 수 있지만, 그 값이 자동으로 Codex나 GitHub 저장소에 들어가지는 않습니다.

현재 흐름:
1. 모바일에서 자기채점과 메모를 작성합니다.
2. 메모를 복사합니다.
3. Codex에 붙여넣습니다.
4. Codex가 오류 분석, 복습 계획, 다음 학습 방향에 반영합니다.

짧은 답변:

> 현재는 수동 피드백 루프입니다. 자동화를 위해서는 모바일 입력값을 서버, GitHub Issue, Form, Database 중 하나로 보내는 연결이 추가되어야 합니다.

### Q4. NotebookLM이나 자료작성 Tool과 Codex가 바로 연계 가능한가?

답변:

현재 가장 안정적인 방식은 파일 기반 연계입니다.

흐름:
1. Codex가 Markdown, PDF, PPT 소스 원고를 만듭니다.
2. 사용자가 NotebookLM에 업로드합니다.
3. NotebookLM이 소스를 바탕으로 발표자료, 요약, Q&A를 생성합니다.

짧은 답변:

> 지금은 Codex가 소스를 만들고 NotebookLM이 그 소스를 읽는 방식이 가장 안정적입니다. 직접 연동은 별도 커넥터나 API가 필요합니다.

## 최종 발표 내러티브

Opening:

> 언어학습에서 가장 어려운 것은 공부할 마음보다 시작까지의 마찰입니다. 오늘 무엇을 할지 고르고, 자료를 찾고, 기록을 남기는 과정이 반복되면 학습은 금방 끊깁니다.

Problem:

> 저는 중국어 HSKK 말하기를 준비하면서, 언제든 휴대폰에서 바로 열 수 있는 개인 학습 루틴이 필요했습니다.

Solution:

> 그래서 Codex와 Harness를 이용해 매일 학습 콘텐츠를 만들고, GitHub Pages에 모바일 화면으로 배포하고, 복습 루프까지 관리하는 구조를 만들었습니다.

Demo:

> 지금 보시는 영상은 별도 앱을 설치한 것이 아니라, GitHub Pages를 홈 화면 바로가기로 등록해서 앱처럼 사용하는 장면입니다.

Harness:

> 뒤에서는 여러 Agent가 역할을 나누어 일합니다. 커리큘럼을 설계하고, 오늘의 말하기 레슨을 만들고, 복습 일정을 관리하고, 모바일 페이지를 다시 빌드합니다.

Closing:

> 이 사례의 핵심은 AI가 답을 한 번 주는 것이 아닙니다. 매일 반복되는 개인 학습 운영을 AI와 자동화가 대신 준비하고, 사람은 실제 말하기 연습에 집중하도록 만든 것입니다.
