# HSKK AI Trainer 시연영상 제작 요청 스크립트

## 1. 제작 목적

다른 프로젝트의 Harness에 아래 두 개의 참고 영상을 전달해, 보고자료 PPT 유첨용 HSKK AI 학습 루프 시연영상을 제작한다.

- 참고 영상 1: `C:\Users\swims\OneDrive\문서\NotebookLM\시연영상1.mp4`
- 참고 영상 2: `C:\Users\swims\OneDrive\문서\NotebookLM\시연영상2.mp4`

최종 영상은 사용자가 모바일 홈 화면 바로가기에서 GitHub Pages 학습 페이지에 진입하고, 오늘 레슨을 확인한 뒤, 자체 피드백 메모와 AI Trainer 녹음 평가까지 이어지는 전체 흐름을 보여준다.

핵심 메시지:

> 내가 원할 때 모바일에서 HSKK 레슨을 확인하고, 바로 말하고, AI 피드백으로 다시 연습하는 개인화 학습 루프

## 2. 참고 영상별 내용 설명

### 시연영상1: GitHub Pages 레슨 학습 흐름

영상은 아이폰 홈 화면에서 시작한다. 사용자는 홈 화면에 추가해 둔 GitHub Pages 바로가기를 통해 HSKK Daily 학습 페이지에 들어간다.

이후 `Lesson 10: 공부 습관` 레슨 화면이 나타난다. 화면에는 오늘의 학습 주제, 설명, 관찰 순서, HSKK Part 2 그림 묘사형 문장이 표시된다. 각 학습 카드에는 한자, 병음, 한국어 뜻이 함께 제공되어 모바일에서 바로 읽고 말하기 연습을 할 수 있다.

중간 이후에는 자기채점/메모 영역이 나타난다. Vocabulary, Fluency and timing, Task completion 같은 항목과 `음성 transcript / 메모` 입력 영역이 보인다. 이 부분은 사용자가 학습 후 스스로 부족한 점을 남기는 장면으로 사용한다.

후반부에서는 `완료하고 다음 레슨 보기`를 통해 다음 레슨으로 넘어간다. 이어서 `Lesson 11: 자기소개와 하루 습관` 화면이 보이고, 하단에 `녹음하고 AI 평가 받기` 버튼이 나타난다. 이 버튼은 레슨을 본 직후 AI Trainer로 이동해 바로 녹음 평가를 받을 수 있음을 보여주는 핵심 장면이다.

### 시연영상2: AI Trainer 녹음 평가 흐름

영상은 `녹음하고 AI 평가 받기` 버튼을 누르는 장면에서 시작한다. 이후 HSKK AI Trainer 화면으로 전환된다.

Trainer 화면 상단에는 `확인`, `녹음`, `재생`, `평가` 단계가 표시되어, 사용자가 지금 어느 단계에 있는지 알 수 있다. `오늘 레슨`과 `예상문제 Test` 선택 영역이 있으며, `예상문제 Test`에서는 Lesson 11과 연결된 예상문제가 표시된다.

예상문제 카드에는 다음과 같은 구성이 보인다.

- 레슨 기반 문제 라벨: `Lesson 11 · 질문 1`
- 문제 한자: `你每天什么时候学习汉语？`
- 성조 병음: `Nǐ měitiān shénme shíhou xuéxí Hànyǔ?`
- 한국어 뜻: `매일 언제 중국어를 공부하나요?`
- 제한 시간: `60s`

사용자는 문제를 눈으로 보면서 녹음을 시작한다. 녹음 중에는 `말하는 중입니다. 끝나면 정지를 누르세요.`라는 안내가 보인다. 녹음이 끝나면 `내 녹음 들어보기` 오디오 플레이어가 표시되어 본인의 답변을 직접 확인할 수 있다.

이후 `평가하기`를 실행하면 STT와 AI 평가가 진행된다. 평가 중에는 원어민 발음 비교 카드가 표시되고, 평가 완료 후에는 `원어민 발음 듣기`, `교정 답변 발음 듣기`, `AI 코칭 리포트`가 순차적으로 보인다.

마지막 화면에서는 AI 코칭 리포트의 `다음 연습 추천`과 `교정 답변`이 보인다. 예시 교정 답변은 `我每天晚上八点学习汉语。每天我学习三十分钟。`처럼 완전한 중국어 문장으로 제시된다. 하단에는 `다시 말하기`, `레슨 계속 보기` 버튼이 있어 학습자가 다시 녹음하거나 원래 레슨으로 돌아갈 수 있다.

## 5. 편집 지시사항

- 최종 영상은 1분 30초 이내로 제작한다.
- 세로형 9:16 모바일 시연 포맷을 유지한다.
- 자막은 필수이며, 화면 하단에 배치하되 주요 버튼과 오디오 플레이어를 가리지 않는다.
- 영상 속 전환이 어색한 부분은 빠른 컷 또는 페이드로 정리한다.
- 브라우저 주소창이나 불필요한 상태 표시가 거슬리면 살짝 크롭하거나 블러 처리한다.
- 화면 터치 위치는 짧은 원형 하이라이트로 강조해도 좋다.
- 배경음악은 있더라도 낮게 깔고, 실제 화면 조작 이해를 방해하지 않게 한다.
- 음성 내레이션은 선택 사항이다. 기본 산출물은 무음 + 자막 버전을 우선 제작한다.
- 기능 표현은 “공식 HSKK 채점”이 아니라 “연습용 AI 코칭/피드백”으로 표현한다.

## 6. Harness 요청용 최종 프롬프트

아래 문장을 다른 프로젝트의 Harness에 그대로 전달한다.

```text
첨부한 두 개의 세로형 모바일 시연영상(시연영상1, 시연영상2)을 참고해 1분 30초 이내의 PPT 유첨용 시연영상을 제작해줘.

영상의 핵심 스토리는 “모바일 홈 화면 바로가기에서 GitHub Pages HSKK 레슨을 열고, 오늘 레슨을 확인한 뒤, 자체 피드백 메모를 남기고, 같은 레슨 주제로 AI Trainer에서 녹음 평가를 받는 학습 루프”야.

반드시 포함할 장면은 다음 순서야.
1. 모바일 홈 화면에서 HSKK 학습 페이지 진입
2. 오늘 레슨에서 한자, 병음, 한국어 뜻 확인
3. 자기채점/음성 transcript 메모 영역 확인 및 자체 피드백 메모 표현
4. 다음 레슨 또는 레슨 하단의 ‘녹음하고 AI 평가 받기’ 버튼 확인
5. HSKK AI Trainer 이동
6. ‘예상문제 Test’에서 방금 본 레슨 기반 질문 표시
7. 문제를 보면서 녹음
8. 내 녹음 들어보기
9. 원어민 발음 듣기와 교정 답변 발음 듣기
10. AI 코칭 리포트의 다음 연습 추천과 교정 답변 확인

자막은 필수로 넣어줘. 자막 문구는 아래를 순서대로 사용해줘.
- 홈 화면 바로가기에서 오늘의 HSKK 학습을 시작합니다
- 한자, 병음, 뜻을 보며 오늘 말할 내용을 먼저 확인합니다
- 학습 후에는 스스로 부족한 점을 메모해 다음 연습 포인트로 남깁니다
- 레슨을 본 직후, 같은 주제로 바로 녹음 평가로 이동합니다
- 예상문제 Test도 방금 본 레슨 주제로 자동 연결됩니다
- 문제를 보면서 답변을 녹음합니다
- 평가 전에 내 발음을 먼저 들어볼 수 있습니다
- STT 결과와 교정 답변을 모범 발음으로 다시 들어봅니다
- 연습용 AI 피드백으로 다시 말할 포인트를 확인합니다

최종 영상은 세로형 9:16, 75~90초 권장, 무음+자막 버전을 기본으로 만들어줘. 음성 내레이션은 선택 산출물로 분리해도 돼. 화면 속 버튼과 핵심 문장을 가리지 않도록 자막을 배치하고, 공식 HSKK 채점처럼 보이지 않게 ‘연습용 AI 피드백’ 표현을 사용해줘.
```
